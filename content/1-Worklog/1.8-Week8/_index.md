### Week 8 Objectives:
* Protect web applications against common security exploits using AWS WAF.
* Organize and manage large-scale AWS resources efficiently using Tags and Resource Groups.
* Implement the IAM principle of least privilege by controlling EC2 access through Resource Tags.
* Set up a monitoring and data visualization system for performance metrics using Grafana.

### Tasks to be carried out this week:
| Day | Task | Start Date | Completion Date | Reference Material |
| :--- | :--- | :--- | :--- | :--- |
| 2 | **AWS WAF:** Introduction to AWS Web Application Firewall. Set up custom rules to mitigate common security vulnerabilities (e.g., SQL Injection, Cross Site Scripting from the OWASP Top 10). | 08/06/2026 | 08/06/2026 | https://000026.awsstudygroup.com/1-introduction/ |
| 3 | **Tags & Resource Groups:** Practice assigning key-value metadata (Tags) to resources. Create Resource Groups based on Tag queries to manage and automate tasks in bulk. | 09/06/2026 | 09/06/2026 | https://000027.awsstudygroup.com/1-introduction/ |
| 5 | **Manage EC2 Access with IAM & Tags:** Configure IAM Policies and Roles with conditional statements. Grant permissions to an EC2 Administrator based on specific Resource Tags to decentralize administration. | 11/06/2026 | 11/06/2026 | https://000028.awsstudygroup.com/1-introduction/ |
| 6 | **Monitoring with Grafana:** Install the open-source Grafana software on a Linux EC2 instance. Query, visualize, and set up alerts from massive metric data streams to monitor AWS resources. | 12/06/2026 | 12/06/2026 | https://000029.awsstudygroup.com/1-introduction/|

### Week 8 Achievements:
* **Application Security (AWS WAF):** Capable of establishing a defense-in-depth system for web applications, proactively blocking/allowing malicious HTTP requests before they reach the system.
* **Resource Organization (Tags & Resource Groups):** Mastered the skill of categorizing resources by purpose, owner, or environment, thereby optimizing the management and operation of thousands of resources simultaneously.
* **Access Control (IAM):** Successfully implemented a decentralized administration model and the principle of least privilege. Strictly controlled the creation and management of EC2 resources through Tag-bound conditions.
* **Visual Monitoring (Grafana):** Successfully built an independent monitoring system, integrating Grafana to chart and analyze the health status of the application infrastructure in real-time.
